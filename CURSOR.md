# CURSOR.md — контекст проекта для агента

Документ описывает **реальный дипломный проект** студента: сайт компании «Дело в шляпе» (детские праздники) с личным кабинетом, бронированием, админкой и Telegram-ботом. При любых правках ориентируйся на **уже принятый в репозитории стиль**, а не на «идеальную архитектуру».

---

## 1. Принципы работы с кодом (обязательно)

1. **Простота важнее всего.** Код должен быть понятен студенту и преподавателю: прямолинейная логика, без лишних абстракций, фреймворков и «умных» паттернов, если это явно не нужно задачей.
2. **Не рефакторить существующий код** без явной просьбы в промпте. Менять только то, что нужно для новой функции или исправления бага.
3. **Минимальный дифф:** не переписывай соседние файлы, не «улучшай» стиль по всему проекту, не удаляй комментарии без причины.

---

## 2. Среда запуска

- **OpenServer** — веб-сервер и PHP; сайт открывается как локальный домен/папка домена (как настроено у студента).
- **База данных:** **MySQL**, управление через **phpMyAdmin** (имя БД в проекте: `Delovshlyape`).
- Подключение к БД в PHP: `includes/db.php` — объект `mysqli` в переменной **`$link`** (хост `localhost`, пользователь `root`, пароль пустой — как в файле; при переносе меняется только там или в настройках окружения).

---

## 3. Технологии по областям

| Область | Технологии |
|--------|------------|
| Сервер и страницы | **PHP** (в т.ч. `session_start()`, `mysqli_*`, `require_once`, `include`) |
| Фронтенд | **HTML/CSS** в `style/`, **ванильный JavaScript** (`fetch`, `FormData`, DOM), без обязательного единого бандлера |
| Админка | `admin.php` + `adminmanage/admin.js` + эндпоинты в `adminmanage/*.php` |
| Бот | **Python 3** в каталоге `bot/`: `python-telegram-bot`, `mysql.connector`, `python-dotenv` (переменные в `.env`) |
| БД | **MySQL** — те же таблицы, что использует сайт (`users`, `programs`, `bookings`, и т.д.) |

---

## 4. Структура проекта (ориентир)

- **`includes/`** — общие куски: `db.php`, `header.php`, `footer.php`, формы и карточки.
- **Корень** — публичные страницы: `index.php`, `programs.php`, `booking.php`, `profile.php`, `auth.php`, `admin.php` и обработчики (`process_booking.php`, `booking-handler.php`, и т.п.).
- **`adminmanage/`** — AJAX/POST-обработчики для админки; ответы часто в формате **JSON** (`Content-Type: application/json`, `json_encode(['success' => true])` и т.д.).
- **`js/`** — скрипты для публичной части (`script.js`, `reviews.js`, `preloader.js` и др.).
- **`style/`** — общие и страничные стили (`general.css` и др.).
- **`images/`**, **`img/`** — медиа (в т.ч. `img/programs/` для обложек программ, галерея может использовать `images/gallery/`).
- **`elements/`** — встраиваемые блоки авторизации/логина.
- **`bot/`** — только бот: `bot.py`, `config.py`, `database.py`; токен и параметры БД через **`.env`**, не хардкодить секреты в репозиторий.

---

## 5. Соглашения PHP

- Подключение БД: `require_once 'includes/db.php';` или `require_once '../includes/db.php';` из подпапок (путь зависит от расположения файла).
- Запросы: **`mysqli_query`**, **`mysqli_fetch_assoc` / `mysqli_fetch_all`**, экранирование строк через **`mysqli_real_escape_string($link, ...)`** там, где так уже сделано; ID приводить к **`(int)`** где уместно.
- Сессии: **`$_SESSION['client_id']`** для клиента; **`$_SESSION['is_admin'] === 1`** для доступа в `admin.php`. Проверки доступа — как в существующих файлах (редирект + `exit`).
- Новые обработчики в стиле проекта: простой скрипт сверху вниз, без лишних классов, если задача маленькая.

---

## 6. Соглашения JavaScript

- В админке: **`adminmanage/admin.js`** — модалки, **`fetch` + `FormData`**, разбор **`response.json()`**, при успехе часто **`location.reload()`**.
- Глобальные данные с сервера в админке передаются через встроенный в `admin.php` блок **`<script>`** с переменными вроде массивов программ/типов (как уже сделано) — новые сущности подключать так же, если нужен список на клиенте.
- Без принудительного перехода на TypeScript/модули, если пользователь не просит.

---

## 7. Соглашения Python (`bot/`)

- Зависимости и запуск — так, как настроено у студента (виртуальное окружение при необходимости).
- **`config.py`**: `TOKEN`, `DB_CONFIG`, тексты `COMPANY_INFO` / `CONTACTS`.
- **`database.py`**: класс **`Database`**, методы с **`mysql.connector`**, параметризованные запросы **`%s`**, закрытие соединения в **`finally`**.
- **Пароли пользователей** в боте проверяются так же, как в PHP: двойной **MD5** + **salt** из таблицы `users` (см. существующий код в `database.py` — не менять алгоритм без согласования с PHP-регистрацией).

---

## 8. Добавление нового функционала (чеклист)

1. Понять, к какой части относится задача: публичный сайт / админка / бот / БД.
2. Найти **аналогичный** уже работающий фрагмент (та же папка, похожий обработчик) и повторить структуру.
3. Для админки: при необходимости новый эндпоинт в **`adminmanage/`** + вызов из **`admin.js`** + элемент интерфейса в **`admin.php`** / **`adminmanage/admin.css`**.
4. Для БД: миграции в проекте могут быть ручными через phpMyAdmin — при описании изменений можно кратко указать нужные поля/таблицы.
5. После правок: проверить пути **`require_once`** относительно нового файла.

## 9. Язык интерфейса и комментариев

- Интерфейс и сообщения пользователю — **русский**, как на сайте.
- Комментарии в коде — по необходимости, **кратко**; не раздувать документацию без запроса.

---

## 10. Чего не делать по умолчанию

- Не внедрять Composer-фреймворки, ORM, Docker, CI — пока проект учебный и на PHP без них.
- Не переписывать весь сайт на ООП/роутер «ради порядка».
- Не менять имена таблиц/полей в SQL без необходимости и без обновления всех мест использования (PHP + JS + бот).

Если в промпте указано иное (например, «отрефактори» или «переведи на PDO») — следовать промпту.
